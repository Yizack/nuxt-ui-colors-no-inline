# nuxt-ui-colors-no-inline

Nuxt module that ensures `nuxt-ui-colors` CSS generated by [Nuxt UI](https://github.com/nuxt/ui) respects the Nuxt `inlineStyles` config when disabled for production builds.

> [!NOTE]
> This module works only with `inlineStyles: false` in your Nuxt config. Without this setting, it defers to Nuxt UI's default behavior.

## Quick setup

1. Add `nuxt-ui-colors-no-inline` dependency to your project

```bash
# Using pnpm
pnpm add -D nuxt-ui-colors-no-inline

# Using yarn
yarn add --dev nuxt-ui-colors-no-inline

# Using npm
npm install --save-dev nuxt-ui-colors-no-inline
```

2. Add the module in your `nuxt.config.ts` after `@nuxt/ui`:

```js
export default defineNuxtConfig({
  modules: [
    '@nuxt/ui',
    'nuxt-ui-colors-no-inline'
  ],
})
```

## Why?

By default, Nuxt UI uses a [colors plugin](https://github.com/nuxt/ui/blob/v4/src/runtime/plugins/colors.ts) to inject a `<style id="nuxt-ui-colors">` tag in the `<head>` of your pages, even when `inlineStyles` is disabled.

If you're particular about clean HTML with no inline styles, this module is for you. It removes Nuxt UI's colors plugin and generates a separate CSS file that's included in your app's main CSS entry instead, fully respecting the `inlineStyles` config.
